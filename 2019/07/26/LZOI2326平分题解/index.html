2326: 平分

时间限制: 1 Sec 内存限制: 128 MB
提交: 62 解决: 15
[提交][状态][题解]
题目描述

Bill收集了一些价值分为6个等级的矿石，现在他希望把这些矿石按价值平分成两堆，但是他不知道是否能实现，于是请你帮忙判断是否可行。
输入

第一行一个整数T（T<=10），为测试数据组数。

下面T行，每行6个数字(int范围)，表示价值等级从1~6的矿石数目。

输出

T行，如果能够平分成两份则该行输出“YES”，否则输出“NO”。
样例输入

2
1 0 1 2 0 0
1 0 0 0 1 1

样例输出

NO
YES

同样的，我们还是先理解一下题意：

其实很简单，就是能不能把一坨1,2,3，4,5,6平均分成两份，使两份和一样

那么再转换一下，假设所以1,2,3,4,5,6的和为2*ss 那么就相当于要刚好凑一个ss

一种最简单情况就是所有的和不是2的倍数，那显然是不可以的

然后我们进入正题：判断能否凑ss

我们考虑从大数值往小数值来凑

什么意思呢？？比如说ss，我们先拿6去凑，结果发现ss=，那么再把b用5凑，以此类推

这个正确性是可以证明的。因为假设有6不够凑，那么可以用3或2或1，而且1,2,3,4,5,6数值差只有1，所以精度肯定没有问题

那就是一道贪心了嘛。能用大的就用大的
